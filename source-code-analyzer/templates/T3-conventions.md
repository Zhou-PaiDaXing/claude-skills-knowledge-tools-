# T3: 代码规范分析

## 分析框架

### 1. 命名规范
- 文件/目录：kebab-case? PascalCase?
- 变量/函数：camelCase? snake_case?
- 布尔值：isX/hasX/shouldX?
- 类/接口：前缀规则？

### 2. 文件组织
- 目录结构模式（展示模式而非列举文件）
- 单文件内容边界、导入导出风格

### 3. 错误处理模式
展示项目中统一的错误处理代码模式。

### 4. 异步模式
展示项目中统一的异步处理代码模式。

### 5. 测试规范
文件位置、命名、结构模式。

### 6. 规范速查卡
```
┌─────────────────────────────┐
│       规范快速参考            │
├─────────────────────────────┤
│ 文件:     ...               │
│ 组件:     ...               │
│ 函数:     ...               │
│ 常量:     ...               │
│ 导出:     ...               │
│ 测试:     ...               │
└─────────────────────────────┘
```
前3条值得采纳的规范。
