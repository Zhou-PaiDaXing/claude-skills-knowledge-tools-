# T1: 架构模式分析

## 分析框架

### 1. 架构风格识别
判断项目采用哪种架构：单体/模块化单体/微服务/分层/整洁/六边形/事件驱动/插件式
用代码库证据支持判断。

### 2. 模块边界
- 什么定义了模块边界？
- 模块之间如何通信？
- 如何防止耦合？

### 3. 架构图
绘制文本架构图，展示核心模块和数据流：
```
[模块A] → [模块B] → [模块C]
    ↓
[模块D]
```

### 4. 扩展点
系统可以在哪里无需修改即可扩展？插件接口、钩子点、策略模式。

### 5. 权衡分析
| 决策 | 收益 | 代价 |
|------|------|------|

### 6. 可借鉴
前3个值得应用到其他项目的架构想法。
