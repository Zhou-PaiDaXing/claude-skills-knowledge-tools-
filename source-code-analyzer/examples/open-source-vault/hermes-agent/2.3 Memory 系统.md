# 2.3 Memory 系统 — 冻结快照 + 跨会话 cache

> 📚 来源:[[hermes-agent-learning-plan#2.3 Memory 系统 — 冻结快照 + 跨会话 cache|学习计划 · 阶段二 · 2.3]]
> 🧭 相关:[[2.1 Tool Registry 自注册模式]] · [[2.4 5 阶段上下文压缩]] · [[3.1 三层闭环]]

---

## 核心思路

`_system_prompt_snapshot` 在会话开始时**冻结**(snapshot),运行时写入新的 memory 不会破坏当前的 prompt cache——下一轮会用新快照,本轮仍然命中 cache。

> **关键设计**:把"写入 memory"与"使用 memory 渲染 system prompt"在时间上解耦,以此换取跨会话 1 小时 prompt cache 的高命中率。

---

## 代码入口

- `agent/memory_manager.py` — 双文件结构 `MEMORY.md` + `USER.md`,原子写 + fcntl 锁
- `agent/prompt_caching.py` — 1 小时 cache TTL,跨会话保留(Claude / OpenRouter / Nous Portal)
