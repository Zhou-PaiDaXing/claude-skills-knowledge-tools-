# 2.1 Tool Registry — 自注册模式

> 📚 来源:[[hermes-agent-learning-plan#2.1 Tool Registry — 自注册模式|学习计划 · 阶段二 · 2.1]]
> 📋 项目简介:[[hermes-agent|hermes-agent 项目概览]]
> 🧭 同章节相关:[[2.2 Provider-Transport 双层抽象]]

---

## 核心思想

> 工具模块自己负责注册自己,注册中心只负责发现和导入。实现"开闭原则"——新增工具只需添加文件,无需修改注册中心。

---

## 设计模式

| 模式 | 说明 |
|------|------|
| **注册表模式 (Registry Pattern)** | 所有工具注册到统一的 `ToolRegistry` 单例 |
| **自注册模式 (Self-Registration Pattern)** | 工具模块在顶层调用 `registry.register()` 完成注册 |
| **依赖注入模式 (Dependency Injection)** | 工具模块声明依赖(`check_fn`、`requires_env`),运行时验证 |

---

## 工具加载过程

```
启动时
    ↓
model_tools.py 调用 discover_builtin_tools()
    ↓
discover_builtin_tools() 扫描 tools/ 目录
    ↓
对每个 .py 文件进行 AST 分析,检查是否包含 registry.register() 调用
    ↓
对符合条件的文件,使用 importlib.import_module() 动态导入
    ↓
模块导入时,模块顶层代码自动执行
    ↓
registry.register() 被调用,工具注册到 ToolRegistry 单例中
    ↓
工具现在可以通过 registry.get_entry(name) 获取,通过 registry.dispatch(name, args) 调用
```

## AST 分析原理

```python
# registry.py 中的 AST 分析逻辑
def _is_registry_register_call(node: ast.AST) -> bool:
    """检查节点是否为 registry.register(...) 调用"""
    # AST 结构: registry.register(...)
    # └── ast.Expr (表达式语句)
    #     └── ast.Call (函数调用)
    #         └── ast.Attribute (属性访问)
    #             ├── ast.Name(id="registry")
    #             └── attr: "register"
    return (
        isinstance(node, ast.Expr) and
        isinstance(node.value, ast.Call) and
        isinstance(node.value.func, ast.Attribute) and
        node.value.func.attr == "register" and
        isinstance(node.value.func.value, ast.Name) and
        node.value.func.value.id == "registry"
    )
```

## 工具模块示例 (`terminal_tool.py`)

```python
# 1. 导入 registry 单例
from tools.registry import registry

# 2. 定义工具实现
def _handle_terminal(args: dict, **kwargs) -> str:
    # 工具实现逻辑...
    pass

# 3. 定义工具 schema
TERMINAL_SCHEMA = {
    "name": "terminal",
    "description": "执行终端命令",
    "parameters": {...}
}

# 4. 定义依赖检查函数
def check_terminal_requirements() -> bool:
    """检查终端工具依赖是否满足"""
    return True

# 5. 模块顶层的自注册调用
registry.register(
    name="terminal",
    toolset="terminal",
    schema=TERMINAL_SCHEMA,
    handler=_handle_terminal,
    check_fn=check_terminal_requirements,
    emoji="💻",
    max_result_size_chars=100_000,
)
```

## 设计思想

| 原则 | 体现 |
|------|------|
| **单一职责 (SRP)** | 每个工具模块只负责一个工具的实现 |
| **开闭原则 (OCP)** | 新增工具只需添加文件,无需修改现有代码 |
| **依赖倒置 (DIP)** | 工具模块依赖抽象的 `registry` 接口,而非具体实现 |
| **接口隔离 (ISP)** | `registry.register()` 提供灵活参数,工具按需使用 |
| **里氏替换 (LSP)** | 工具可通过 `override=True` 替换现有实现 |

## 可借鉴点

1. **插件系统**:插件模块自注册,主程序动态发现
2. **命令注册**:命令模块自注册,CLI 动态发现
3. **中间件系统**:中间件模块自注册,框架动态加载
4. **事件处理器**:事件处理器模块自注册,事件总线动态发现
5. **日志处理器**:日志处理器模块自注册,日志框架动态加载

## 代码入口

- **注册中心**:`tools/registry.py`
  - `ToolRegistry` 类:工具注册表单例
  - `discover_builtin_tools()`:AST 扫描发现工具模块
  - `_is_registry_register_call()`:AST 节点检查
- **工具示例**:`tools/terminal_tool.py:2370` (`registry.register()` 调用)
- **工具调用**:`model_tools.py` (`handle_function_call()` 分发工具调用)
